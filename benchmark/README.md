# codemap-search 검색 품질 벤치마크

이 폴더는 `codemap-search`가 코드 에이전트의 검색 품질, 토큰 사용량, 도구 호출 수에 미치는 영향을 같은 조건에서 비교하기 위한 고정 벤치마크다. 2026년 7월 측정에 사용한 Directus 소스, 14개 과제, 실행·채점·집계 코드를 보존한다.

실제 실행 순서와 명령은 [상세 실행 안내서](RUNBOOK.md)를 따른다. 하네스 내부 구조와 파일별 역할은 [하네스 안내](harness/README.md)를 참고한다.

### B1과 B2

| 조건 | OpenCode 기본 도구 | `codemap-search` MCP | 사용 강제 |
|---|---:|---:|---:|
| B1 | 사용 | 없음 | 해당 없음 |
| B2 | B1과 동일 | `codemap-search` 하나만 추가 | 강제하지 않음 |

B1과 B2는 모델, 질문, 소스, 기본 도구, 권한, 제한이 같아야 한다. 허용되는 차이는 B2의 `mcp.codemap_search` 설정뿐이다.

```text
benchmark/

└── harness/              # 실행·격리·복구·채점 코드와 JSON 계약
```

`corpus/directus`는 위 커밋의 Git 트리에서 직접 만든 4,353개 파일 스냅샷이다. 중첩 `.git`, 로컬 `.codemap`, 설치된 의존성은 포함하지 않는다.

## 저장소에 포함하지 않는 자료

다음 자료는 크기가 크거나 인증 정보를 포함하거나 실행 환경에 종속되므로 커밋하지 않는다.

- OpenCode 실행 파일과 Ollama Cloud 인증 자료
- B2 제품 소스 복사본과 두 번의 Rust 빌드 결과
- Directus의 고정 `.codemap` 색인
- 실제 대화, 자동 지표, 채점 결과, 임시 작업 폴더
- Python 캐시

따라서 이 저장소를 복제한 직후에는 채점·분석 코드의 자체검사는 가능하지만, 84개 외부 모델 세션을 바로 실행할 수는 없다. 빠진 자료와 정식 실행 가능 조건은 [상세 실행 안내서의 준비 상태](RUNBOOK.md#1-현재-준비-상태)에서 확인한다.

## 결과를 해석할 때 지킬 원칙

- B1과 B2는 반드시 같은 14개 과제와 같은 3회 반복으로 비교한다.
- 실패한 시도를 삭제하지 않는다. 하네스가 원자료와 실패 이유를 보존하고, 집계 가능한 최종 시도만 평균에 포함한다.
- 제공자·네트워크·인증의 일시적 실패만 같은 자리를 다시 실행한다.
- 시간 초과, 모델 단계 제한, 출력 제한은 미리 정한 유효한 관찰 결과이므로 자동으로 성공 결과로 바꾸지 않는다.
- 코드, 질문, 설정, 모델, 색인, 실행 파일이 바뀌면 같은 실행 세대를 이어 쓰지 않는다.
- B2에서 MCP 사용을 강제하지 않는다. MCP를 호출하지 않은 사실도 검색 행동 차이의 일부다.
- 토큰과 도구 호출 수는 OpenCode 원자료에서 추출하며 사람이 임의로 채우지 않는다.
