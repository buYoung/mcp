Let me look at the specific helper files that are copied between apps and check their dependency declarations.
이 코드베이스의 실제 데이터를 분석한 결과를 정리합니다.

---

## 핵심 구조

이 모노레포는 두 개의 TypeScript 앱(`@buyong-mcp/scout`, `@buyong-mcp/acp-bridge`)과 하나의 Rust 바이너리(`codemap-search`)로 구성됩니다. 두 TypeScript 앱 모두 **번들러 없이 순수 `tsc`**로 빌드하여 `dist/*.js` 개별 파일을 출력합니다. `scout`는 `mcp-server`에서 헬퍼를 **복사하여** 독립적으로 유지합니다(`scout/AGENTS.md:47` — "Copied helpers stay local").

## 동작하는 해석 경로

`dist/startup/binary-availability.js` 같은 생성 출력 파일이 `@modelcontextprotocol/sdk`를 import할 때, Node.js 모듈 해석은 **파일 위치에서 디렉토리 트리를 위로 올라가며** `node_modules/`를 찾습니다:

```
apps/scout/dist/startup/binary-availability.js
  → apps/scout/dist/startup/node_modules/ 없음
  → apps/scout/dist/node_modules/ 없음
  → apps/scout/node_modules/ 없음 (pnpm은 여기에 직접 설치하지 않음)
  → [workspace-root]/node_modules/ ← 여기서 해석됨
```

pnpm은 `apps/scout/package.json`에 선언된 의존성을 **workspace 루트 `node_modules/`에 심링크로 호이스팅**합니다. 따라서 헬퍼의 own `package.json`에 해당 패키지가 선언되어 있지 않아도, 전체 릴리스가 이미 그 패키지를 workspace 루트에 가져왔으므로 빌드와 실행이 성공합니다.

## 잘못된 버전이 조용히 바인딩되는 경우

`pnpm-lock.yaml`의 실제 데이터가 핵심을 보여줍니다:

```yaml
# 앱들의 직접 의존성
apps/mcp-server: '@modelcontextprotocol/sdk': 1.29.0(zod@4.4.3)
apps/scout:      '@modelcontextprotocol/sdk': 1.29.0(zod@4.43)

# inspector (mcp-server의 devDependency)의 의존성
@modelcontextprotocol/inspector: '@modelcontextprotocol/sdk': 1.29.0(zod@3.25.76)
```

`@modelcontextprotocol/sdk@1.29.0`은 **zod 버전에 따라 두 개의 별도 인스턴스**로 해석됩니다:
- `(zod@4.4.3)` — 앱들이 사용
- `(zod@3.25.76)` — inspector 계열이 사용

**위험 시나리오**: 헬퍼가 번들러(esbuild 등)에 의해 다른 앱의 컨텍스트에서 해석되면, 그 번들러는 **번들 대상 앱의 `node_modules` 컨텍스트**에서 의존성을 찾습니다. 만약:

1. 앱 A가 `zod: ^4.4.3`을 선언하고
2. 앱 B가 `zod: ^3.x`를 선언하고
3. 헬퍼(A에서 복사됨)가 B와 함께 번들되면

번들러는 B의 컨텍스트에서 `@modelcontextprotocol/sdk`를 해석하여 `zod@3.25.76` 버전을 바인딩합니다. **빌드 에러 없이** 잘못된 피어 링크 버전이 조용히 사용됩니다.

## 근본 원인

이것은 **팬텀 의존성(phantom dependency)** 문제입니다. 헬퍼의 own manifest에 선언되지 않은 패키지가:

- **개발 시**: workspace 루트의 호이스팅/심링크 덕분에 항상 올바른 버전으로 해석됨
- **릴리스 번들 시**: 번들러가 소비자의 컨텍스트에서 해석하므로, 소비자 앱이 다른 버전 범위를 선언하면 잘못된 인스턴스가 선택됨

이 위험을 제거하려면 헬퍼가 사용하는 모든 외부 패키지를 **자신의 own `package.json`에 명시적으로 선언**해야 합니다.
