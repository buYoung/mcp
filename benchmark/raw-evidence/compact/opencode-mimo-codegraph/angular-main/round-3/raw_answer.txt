## 핵심 메커니즘 요약

### 1. 재적용 규칙: `_writeValueAfterRender()` 코얼레싱

`select_control_value_accessor.ts:151-192`의 `_queuedWrite` 플래그가 핵심입니다.

자식 option이 추가·변경·파괴될 때마다 즉시 `writeValue()`를 호출하는 대신 `_writeValueAfterRender()`만 호출합니다. 이 메서드는 `_queuedWrite` 플래그로 여러 호출을 **하나의 렌더 사이클로 합치고(coalescing)**, `afterNextRender({ write })`에서 한 번만 실제 재선택을 수행합니다.

```typescript
_writeValueAfterRender(): void {
  if (this._queuedWrite || this.appRefInjector.destroyed) return;
  this._queuedWrite = true;
  afterNextRender({
    write: () => {
      this._queuedWrite = false;
      this.writeValue(this.value);  // 실제 재선택은 여기서 한 번만
    },
  }, {injector: this.appRefInjector});
}
```

### 2. 왜 자식 변경을 즉시 허용하면 안 되는가 — 세 가지 이유

코드 내 주석(`select_control_value_accessor.ts:153-169`)에서 명시합니다:

| 문제 | 원인 |
|------|------|
| **O(n²) 비교 폭발** | 각 option 추가 시 `writeValue`가 호출되면 `_optionMap` 전체를 순회하며 `compareWith`를 반복 호출 (issue #41330) |
| **Safari/IE11 첫 option 미선택** | option이 개별 렌더링되는 동안 `writeValue`가 실행되면, 첫 option element가 올바르게 deselected되지 않음 (issue #14505) |
| **브라우저 drift (animations)** | 애니메이션으로 option이 DOM에서 지연 제거되는 동안 Angular가 DOM value를 먼저 변경하면, 브라우저가 남은 임의 option을 선택해버림 (issue #18430) |

### 3. 비교 함수와 등록 구조

`_optionMap`은 `Map<string, any>`이며 `_idCounter`로 고유 ID를 발급합니다. 비교 함수 기본값은 `Object.is`이며, `[compareWith]` 입력으로 사용자 정의 comparator를 제공할 수 있습니다.

```typescript
// select_control_value_accessor.ts:224-229
_getOptionId(value: any): string | null {
  for (const id of this._optionMap.keys()) {
    if (this._compareWith(this._optionMap.get(id), value)) return id;
  }
  return null;
}
```

### 4. 자식 option의 지연 경로 (대조: 다중 선택)

**단일 선택** — `_writeValueAfterRender()` (지연):
```typescript
// NgSelectOption.ngValue setter (라인 273-279)
this._select._optionMap.set(this.id, value);
this._setElementValue(_buildValueString(this.id, value));
this._select._writeValueAfterRender();  // 지연만 요청
```

**다중 선택** — `writeValue()` (즉시):
```typescript
// select_multiple_control_value_accessor.ts:235-241
this._value = value;
this._setElementValue(_buildValueString(this.id, value));
this._select.writeValue(this._select.value);  // 즉시 재선택
```

다중 선택에서는 각 option이 독립적으로 선택/해제되므로 즉시 호출이 안전하지만, 단일 선택에서는 하나의 값만 유효하므로 여러 option 변경을 묶어서 처리해야 합니다.

### 결론

Angular는 **후속 렌더(write phase)에서 코얼레싱된 one-shot 재선택**을 통해 두 가지 불변식을 동시에 만족합니다:

1. **비교 함수 O(n²) 방지** — 여러 자식 변경이 `_queuedWrite` 하나로 합쳐져 `writeValue`가 한 번만 실행됩니다.
2. **브라우저 drift 방지** — 실제 DOM value 변경이 렌더 후에 이루어지므로, animations 등으로 인한 지연 제거 동안 브라우저가 임의 option을 선택하는 문제를 회피합니다.
