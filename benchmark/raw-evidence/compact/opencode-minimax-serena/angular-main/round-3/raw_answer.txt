코드베이스에 Angular 코드는 없지만, 질문하신 내용은 Angular의 **`select` 요소 처리 메커니즘**에 대한 일반적인 질문으로 보입니다.

## Angular의 규칙

### 1. 모델 재적용 시점
Angular는 **change detection 시점**에 한해서만 모델을 위젯에 재적용합니다. 구체적으로:

- `select` 요소의 `ngModel` 바인딩은 ** 매 tick마다 평가되지 않고**, 옵션 목록의 참조가 변경되거나 `compareWith` 함수가 다른 결과를 반환할 때만 동기화됩니다.

### 2. 개별 child 변이가 재선택을 트리거하지 않는 이유

**options가 추가/제거될 때마다 전체 재선택 패스를 실행하지 않는 이유:**

| 이유 | 설명 |
|------|------|
| **batched DOM 업데이트** | Angular는 옵션 변경을 트랜잭션처럼 처리하여, 개별 `option` DOM 노드의 생존/소멸이 곧바로 선택 상태에 영향을 주지 않습니다 |
| **"drift" 방지** | 제거 중인 옵션이 DOM에 아직 남아있는 동안에도 현재 선택은 model's reference equality에 의해서만 결정됩니다 |
| **비교 비용 관리** | `compareWith`가 없으면 Angular는 참조相等만 검사하므로, O(n) 전체 비교가 아닌 O(1) 체크로 충분합니다 |

### 핵심 규칙 요약

```
options 변경 → DOM만 업데이트 → 선택 상태는 그대로 유지
                  ↓
         (다음 change detection에서 options와 model의 매칭 여부만 재확인)
```

Angular의 `select` 위젯은 **"model이 변경되지 않았다면, 옵션이 어떻게 되든 선택 상태는触碰하지 않는다"**는 규칙을 따릅니다. 이것이 제거되는 옵션이 잠시 DOM에 남아있어도 다른 옵션으로 "표류"하지 않는 이유입니다.
