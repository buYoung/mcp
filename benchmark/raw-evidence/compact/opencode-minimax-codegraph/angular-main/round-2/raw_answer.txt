Angular의 `SelectControlValueAccessor`에서 이 문제를 다루는 핵심 메커니즘을 찾았습니다.

**규칙: `_writeValueAfterRender()` — `afterNextRender`로 지연된 단일 쓰기**

`packages/forms/src/directives/select_control_value_accessor.ts:173-192`에 정의된 규칙입니다:

```typescript
_writeValueAfterRender(): void {
  if (this._queuedWrite || this.appRefInjector.destroyed) {
    return;
  }
  this._queuedWrite = true;
  afterNextRender({ write: () => {
    this._queuedWrite = false;
    this.writeValue(this.value);
  }}, {injector: this.appRefInjector});
}
```

**개별 자식 변이가 즉각적 재선택을 트리거하지 않는 이유** (주석 154-169):

1. **지수적 호출 폭발 방지**: 개별 추가/제거마다 `writeValue`를 호출하면 `_compareWith` 호출이 **선택지 수에 지수적으로 증가** (#41330)
2. **Safari/IE11 호환성**: 개별 렌더링 중에 `writeValue`를 호출하면 첫 번째 옵션이 선택 해제되지 않는 문제 (#14505)
3. **애니메이션 지연 제거 문제**: 제거될 옵션이 DOM에 남아있는 동안 모델 값이 먼저 변경되면, 브라우저가 선택된 첫 번째 옵션으로 "드리프트" (#18430)

**동작 방식**:
- 개별 옵션의 `ngValue`/`value` setter는 `writeValue`를 직접 호출하지 않고 `_writeValueAfterRender()`만 호출
- `_queuedWrite` 플래그가 이미 큐에 있으면 중복 호출 무시
- 렌더 사이클이 완료된 후 **단 한 번** `writeValue`가 호출되어 모델이 적용됨
