Precise suppression in warming/stale/error states cannot be enforced inside the annotation pipeline.
- Callers of homonym functions (e.g., `User.save` vs. `File.save`) are not separated: `calls_by_name["save"]` returns all call sites for the bare name; there is no per-resolution confirmation that a given call site actually targets the definition currently being annotated.
- `source_unresolved` fallback metric is not tracked; there is no counter recording how often import source resolution fails so its ROI can be measured.

## Desired Outcome (To-Be)
- `apps/codemap-search/src/parser/types.rs` adds `ImportEntry` (with `local_name`, `imported_name`, `source`, `kind`, `range`) and `ImportKind` (`Named`, `Default`, `Namespace`, `Glob`) as specified in design doc §5; `NavigationFile.imports` is populated from `@nav.import.*` captures in the runtime extraction pass.
- `apps/codemap-search/queries/typescript/navigation.scm` (proposed, created in child 02) is extended with `@nav.import`, `@nav.import.source`, `@nav.import.name`, `@nav.import.alias`, `@nav.import.namespace`, `@nav.import.default`, and `@nav.import.glob` captures covering named, aliased, default, and namespace TypeScript/TSX import forms.
- `apps/codemap-search/src/callers/symbols.rs` adds `NavigationIndex` (with `calls_by_name: HashMap<String, Vec<CallSiteAddress>>` and `files_by_path: HashMap<String, usize>`) and `CallSiteAddress` (with `file_index` and `call_index`), plus a builder that derives the index from the snapshot's `NavigationFile.calls` at annotation time. A source-hint candidate lookup helper filters the candidate set to the single resolved file and to exported-only symbols when a source hint is present.
- `apps/codemap-search/src/callers/annotate.rs` is extended with a `resolve_call` path: for each navigation call site, import alias remapping is applied (local call name → `imported_name` + `source_hint` via `ImportEntry`), the 6-step relative source resolution rule is applied, and the candidate set is reduced; only call sites that resolve to the current definition (candidate count == 1, exported filter satisfied) are rendered as precise callers. Homonym caller separation — `User.save` vs. `File.save` — is enforced: a call site for `save` is attributed to `User.save` only when the resolution of that call site matches that specific definition. Budget-exceeded call sites fall back to approximate.
- `apps/codemap-search/src/config.rs` adds the `navigation_callsite_budget` key (default value per design doc §7; existing scalar default + `CONFIG_TEMPLATE` comment + `MIGRATIONS` entry + `ResolvedConfig` field). Reaching the budget during an annotation pass stops precise caller resolution and falls back to the existing approximate caller display.
- `apps/codemap-search/src/tools/search/mod.rs` constructs an `AnnotationRuntimeState { is_warming, has_refresh_error, is_dead_or_stale }` from the engine state checks already present at L386–398, and threads it into `annotate_results`. When any flag is true, the annotation pipeline does not attempt navigation-based precise resolution regardless of whether `NavigationIndex` is populated.
- Relative import source resolution follows the 6-step rule: (1) normalize relative to importing file's directory; (2) non-relative paths fall back; (3) extension-absent sources try `.ts`, `.tsx`, `.js`, `.jsx` in order; (4) no match → try `index.ts`, `index.tsx`, `index.js`, `index.jsx`; (5) exactly one workspace-relative match → use it; (6) zero or two-or-more matches → fall back and record `source_unresolved`.
- `source_unresolved` fallback events increment a counter visible in metrics; `navigation_fallback_reason=source_unresolved` is recorded so ROI can be measured.

## Constraints
- `AnnotationRuntimeState` is introduced by child 02 (it defines the struct with `is_warming`, `has_refresh_error`, `is_dead_or_stale` and threads it into `annotate_results` for callee suppression). Child 04 must NOT redefine or recreate the struct — it reuses the existing definition and extends its usage to the caller direction. `is_warming()` on `EngineSupervisor` (L154 in `src/index/supervisor.rs`) and `Indexer` (L84 in `src/index/indexer.rs`) already exists; use it as the source for `AnnotationRuntimeState.is_warming`.
- `NavigationIndex` must be built at annotation-call time from the snapshot passed into `annotate_results`; it must not be stored inside the snapshot struct itself (self-referential reference hazard noted in design doc §5).
- The 6-step source resolution rule must be applied exactly as specified: steps 3 and 4 try extensions in the order `.ts`, `.tsx`, `.js`, `.jsx`; step 5 requires exactly one match; step 6 falls back on zero or two-or-more — no shortcut.
- Import `source_hint` candidates must enforce `is_exported == true`; same-file candidates do NOT require export — the export filter applies only when crossing a file boundary via source hint.
- `navigation_callsite_budget` budget exhaustion must produce a graceful fall back to the existing approximate caller display, never a partial list of precise callers mixed with approximate entries.
- `CONFIG_VERSION` in `src/config.rs` must be bumped once when all three navigation config keys (`navigation_context_default`, `navigation_callsite_budget`, `navigation_store_references`) are added, and a corresponding `MIGRATIONS` entry (or entries) must be appended so existing repo config files pick up all three new commented keys on the next `mcp` start.
- `navigation_context_default` defaults to off and `navigation_store_references` defaults to false in the first implementation, per design doc §7. Child 04 wires `navigation_callsite_budget` into the caller-direction budget check; `navigation_context_default` and `navigation_store_references` are added as keys here so downstream children (child 05 reads `navigation_context_default`; child 02's `ReferenceSite` deferral is gated by `navigation_store_references`) do not reference nonexistent config.
- Homonym caller separation: the precise attribution check for `User.save` and `File.save` requires that `resolve_call` is run independently for each definition, and only call sites whose resolution result matches that specific definition are attributed. A call site that resolves to `User.save` must not appear in `File.save`'s caller list.
