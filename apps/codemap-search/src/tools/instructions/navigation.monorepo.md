Five read-only navigation tools, available after the `initial_instructions` preflight. Pick by intent, not order.
* `overview`: use first when the user gives a broad or underspecified repo request. No path returns workspace scopes only; ask which scope to use before broad work. If the user explicitly wants the whole repo, treat `all` / `전체` as a repo-wide scope. `overview <scope>` narrows and sets the active scope for later `search`.
* `search`: use for symbols, definitions, concepts, quoted strings, errors, config defaults, or 'where is X done?' questions after scope is clear. It uses the active workspace scope selected by `overview <scope>` unless `workspace_scope` overrides it; pass `workspace_scope='all'` / `workspace_scope='전체'` for repo-wide search. Top files include compact `match_reason`, ambiguity hints, `read_suggestion`, line snippets, and matched function depth-1 callers/callees. Set `caller_context=false` to disable.
* `grep`: use for exact enumeration: confirmed names, regex, comments, non-code files, or just-edited files with no index lag. More pages include `next_offset`. Not for semantic or identifier-split lookup — use `search`.
* `read`: read file contents after discovery. Prefer `offset`/`limit` windows from `search.read_suggestion` or `overview`, especially for large files. Don't read a whole large file to find one symbol.
* `find`: locate files by glob and confirm exact paths. Finds paths only, not contents — to search code use `search` or `grep`.
Typical monorepo flow: root `overview` → ask/choose workspace scope (or explicit `all`/`전체`) → `overview <scope>` → scoped `search` → `read`.
