Five read-only navigation tools, available after the `initial_instructions` preflight. Pick by intent, not order.
* `overview`: use first for broad or underspecified repo requests. No path returns workspace scopes only; ask scope before broad work unless the user explicitly wants repo-wide `all` / `전체`. `overview <scope>` narrows and sets the active scope for later `search`.
* `search`: use for symbols, definitions, concepts, quoted strings, errors, config defaults, or 'where is X done?' questions after scope is clear. It uses the active `overview <scope>` unless `workspace_scope` overrides it; pass `all` / `전체` for repo-wide search. Top files include `match_reason`, ambiguity hints, `read_suggestion`, snippets, and depth-1 callers/callees. Set `caller_context=false` to disable. Cite snippets directly only when answer and context are visible with no partial, ambiguity, or list-only hints. Otherwise read each needed `read_suggestion` range once; if absent, narrow the query or use `overview`, then read only needed ranges. Do not reread same ranges to confirm.
* `grep`: exact enumeration for confirmed names, regex, comments, non-code, or just-edited files. More pages include `next_offset`; use `search` for semantic or identifier-split lookup.
* `read`: read after discovery. Prefer `offset`/`limit` windows from `search.read_suggestion` or `overview`; don't read a whole large file to find one symbol.
* `find`: locate files by glob and confirm exact paths. Finds paths only; use `search` or `grep` for contents.
Typical monorepo flow: root `overview` → ask/choose scope (or explicit `all`/`전체`) → `overview <scope>` → scoped `search` → answer from sufficient snippets, or `read` only needed ranges.
