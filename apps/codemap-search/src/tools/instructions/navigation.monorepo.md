Five read-only navigation tools, available after the `initial_instructions` preflight. In a ready monorepo, that response already includes the root overview and its selectable scopes; do not immediately repeat a no-argument `overview`. Repeat it only when the initial response says the index is warming or stopped. Pick by intent, not order.
* `overview`: use for broad or underspecified repo requests after the initial root overview, or to narrow a selected scope. Every scope listed in the root overview, including conventional workspaces and top-level source roots such as `api`, `app`, or `sdk`, is valid for `overview <scope>` and `workspace_scope`. Ask scope before broad work unless the user explicitly wants repo-wide `all` / `전체`. `overview <scope>` narrows and sets the active scope for later `search`.
* `grep`: after scope is clear, use first for an exact identifier, quoted literal, config key/default, exact error text, regex, exhaustive enumeration, comments/non-code, or a just-edited file. It reads disk directly; constrain `path` to the chosen workspace. More pages include `next_offset`. Switch to `search` when wording, spelling, or the relevant area is uncertain.
* `search`: after scope is clear, use first for natural-language behavior, ambiguous concepts, mixed symbol-and-concept questions, relationships between files, or 'where is this done?' questions where exact text is unknown. It uses the active `overview <scope>` unless `workspace_scope` overrides it; pass `all` / `전체` for repo-wide search. Top files include `match_reason`, ambiguity hints, `read_suggestion`, snippets, depth-1 callers/callees, and bounded static collection write/read hints when both endpoints are inside the selected scope and direct type identity is provable. Static collection hints cover TypeScript/JavaScript, Go, Rust, Java, Kotlin, Python, and C/C++; confirm every hinted endpoint in source. Set `caller_context=false` to disable call context. Cite snippets directly only when answer and context are visible with no partial, ambiguity, or list-only hints. Otherwise read each needed `read_suggestion` range once; if absent, narrow the query or use `overview`, then read only needed ranges. Do not reread same ranges to confirm.
* `read`: read after discovery. Prefer `offset`/`limit` windows from `search.read_suggestion` or `overview`; don't read a whole large file to find one symbol.
* `find`: locate files by glob and confirm exact paths. Finds paths only; use `search` or `grep` for contents.
Typical monorepo flow: `initial_instructions` (includes ready root overview) → ask/choose scope (or explicit `all`/`전체`) → choose scoped `grep` for known exact text or scoped `search` for meaning/relationships → answer from sufficient snippets, or `read` only needed ranges.
