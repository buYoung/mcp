Use to orient in unfamiliar code or to get a symbol's line range before `read`. No path returns a bounded repo-root map with recursive file/symbol counts per directory plus the first files by path (grouped significant symbol names, no line ranges); pass `format='llms-txt'` for a bounded LLM-friendly text map instead. A folder path narrows to that folder and lists its files' significant symbol names and kinds, without line ranges. A file path lists that file's significant symbols with exact line ranges. To get a symbol range before reading, run `overview`, then `read` that window. Not for locating a named symbol or concept — use `search`; `overview` orients, so don't call it repeatedly as a search substitute.
