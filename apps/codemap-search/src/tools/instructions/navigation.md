Five read-only navigation tools, available after the `initial_instructions` preflight. Pick by intent, not order.
* `search`: use first for symbols, definitions, concepts, quoted strings, errors, config defaults, or 'where is X done?' questions. BM25 + identifier splitting find what exact grep misses. Top files include `match_reason`, ambiguity hints, `read_suggestion`, snippets, and depth-1 callers/callees. Set `caller_context=false` to disable. Cite snippets directly only when answer and context are visible with no partial, ambiguity, or list-only hints. Otherwise read each needed `read_suggestion` range once; if absent, narrow the query or use `overview`, then read only needed ranges. Do not reread same ranges to confirm. Caller/callee attribution is approximate unless marked tree-sitter precise. Use `grep` for exhaustive literals or files edited since last index.
* `grep`: exact enumeration for confirmed names, regex, comments, non-code, or just-edited files. More pages include `next_offset`; use `search` for semantic or identifier-split lookup.
* `overview`: use to orient in unfamiliar code or get symbol line ranges before `read`. No path returns a bounded repo map with compact file/symbol groups. Folder path narrows. File path gives exact symbol ranges. Not for locating a named symbol — use `search`; it orients, so don't call it repeatedly as a search substitute.
* `read`: read after discovery. Prefer `offset`/`limit` windows from `search.read_suggestion` or `overview`; don't read a whole large file to find one symbol.
* `find`: locate files by glob and confirm exact paths. Finds paths only; use `search` or `grep` for contents.
Typical flow: `overview` when unfamiliar → `search` → answer from sufficient snippets, or `read` only needed ranges. Use `grep` for exact enumeration or just-edited files.
