Five read-only navigation tools, available after the `initial_instructions` preflight. Pick by intent, not order.
* `grep`: use first for an exact identifier, quoted literal, config key/default, exact error text, regex, exhaustive enumeration, comments/non-code, or a just-edited file. It reads disk directly and avoids semantic ranking when the requested text is already known. More pages include `next_offset`. If an exact lookup fails, check path/glob/case first; switch to `search` when wording, spelling, or the relevant area is uncertain.
* `search`: use first for natural-language behavior, ambiguous concepts, mixed symbol-and-concept questions, relationships between files, or 'where is this done?' questions where the exact text is unknown. BM25 + identifier splitting find what exact grep misses. Top files include `match_reason`, ambiguity hints, `read_suggestion`, snippets, depth-1 callers/callees, and bounded static collection write/read hints when direct type identity is provable. Static collection hints cover TypeScript/JavaScript, Go, Rust, Java, Kotlin, Python, and C/C++; confirm every hinted endpoint in source. Set `caller_context=false` to disable call context. Cite snippets directly only when answer and context are visible with no partial, ambiguity, or list-only hints. Otherwise read each needed `read_suggestion` range once; if absent, narrow the query or use `overview`, then read only needed ranges. Do not reread same ranges to confirm. Caller/callee attribution is approximate unless marked tree-sitter precise.
* `overview`: use to orient in unfamiliar code or get symbol line ranges before `read`. No path returns a bounded repo map with compact file/symbol groups. Folder path narrows. File path gives exact symbol ranges. Not for locating a named symbol — use `search`; it orients, so don't call it repeatedly as a search substitute.
* `read`: read after discovery. Prefer `offset`/`limit` windows from `search.read_suggestion` or `overview`; don't read a whole large file to find one symbol.
* `find`: locate files by glob and confirm exact paths. Finds paths only; use `search` or `grep` for contents.
Typical flow: choose `grep` for known exact text or `search` for meaning/relationships → answer from sufficient snippets, or `read` only needed ranges. Use `overview` first only when the repository area itself is unfamiliar.
